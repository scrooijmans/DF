# The SciChartPolarSurface Type

The root polar chart view is called the <a href="https://www.scichart.com/documentation/js/v4/typedoc/classes/scichartpolarsurface.html" rel="noopener noreferrer" target="_blank">SciChartPolarSurfaceðŸ“˜</a>. This is the JavaScript chart control you will be adding to your applications wherever you need a **Polar Chart**. You can add more than one SciChartPolarSurface to an HTML page and combine them with [Cartesian SciChartSurface](https://www.scichart.com/documentation/js/v4/2d-charts/surface/scichart-surface-type-overview), you can configure them independently, and you can link them together.

The <a href="https://www.scichart.com/documentation/js/v4/typedoc/classes/scichartpolarsurface.html" rel="noopener noreferrer" target="_blank">SciChartPolarSurfaceðŸ“˜</a> extends <a href="https://www.scichart.com/documentation/js/v4/typedoc/classes/scichartsurface.html" rel="noopener noreferrer" target="_blank">SciChartSurfaceðŸ“˜</a> and inherits the same properties and functions which allow you to configure and control the chart.

You will find information about the [parent SciChartSurface class here](https://www.scichart.com/documentation/js/v4/2d-charts/surface/scichart-surface-type-overview).

Info about the properties and functions available can be found at the <a href="https://www.scichart.com/documentation/js/v4/typedoc/index.html" rel="noopener noreferrer" target="_blank">TypeDoc API Documentation for SciChartðŸ“˜</a>.

## Surface Type Check for Renderable Series, Annotations and Chart Modifiers<a href="https://www.scichart.com/documentation/js/v4/2d-charts/surface/scichart-polar-surface-type/#surface-type-check-for-renderable-series-annotations-and-chart-modifiers" class="hash-link" aria-label="Direct link to Surface Type Check for Renderable Series, Annotations and Chart Modifiers" translate="no" title="Direct link to Surface Type Check for Renderable Series, Annotations and Chart Modifiers">â€‹</a>

When <a href="https://www.scichart.com/documentation/js/v4/typedoc/interfaces/irenderableseries.html" rel="noopener noreferrer" target="_blank">IRenderableSeriesðŸ“˜</a> is attached to <a href="https://www.scichart.com/documentation/js/v4/typedoc/classes/scichartsurface.html" rel="noopener noreferrer" target="_blank">SciChartSurfaceðŸ“˜</a> or to <a href="https://www.scichart.com/documentation/js/v4/typedoc/classes/scichartpolarsurface.html" rel="noopener noreferrer" target="_blank">SciChartPolarSurfaceðŸ“˜</a> a type check is performed, <a href="https://www.scichart.com/documentation/js/v4/typedoc/interfaces/irenderableseries.html#ispolar" rel="noopener noreferrer" target="_blank">renderableSeries.isPolarðŸ“˜</a> property should match the surface type.

When <a href="https://www.scichart.com/documentation/js/v4/typedoc/interfaces/ichartmodifierbase.html" rel="noopener noreferrer" target="_blank">IChartModifierBaseðŸ“˜</a> is attached to the surface a type check is also performed, <a href="https://www.scichart.com/documentation/js/v4/typedoc/interfaces/ichartmodifierbase.html#modifiertype" rel="noopener noreferrer" target="_blank">chartModifier.modifierTypeðŸ“˜</a> property should match <a href="https://www.scichart.com/documentation/js/v4/typedoc/classes/scichartsurface.html#surfacetype" rel="noopener noreferrer" target="_blank">sciChartSurface.surfaceTypeðŸ“˜</a>.

For renderable series and annotations the surface type should be obvious from the name as the polar items have it in the name. For example, [FastLineRenderableSeries](https://www.scichart.com/documentation/js/v4/2d-charts/chart-types/fast-line-renderable-series) and [PolarLineRenderableSeries](https://www.scichart.com/documentation/js/v4/2d-charts/chart-types/polar-line-renderable-series), [CursorModifier](https://www.scichart.com/documentation/js/v4/2d-charts/chart-modifier-api/cursor-modifier/cursor-modifier-overview) and [PolarCursorModifier](https://www.scichart.com/documentation/js/v4/2d-charts/chart-modifier-api/polar-modifiers/polar-cursor-modifier).

A similar check is implemented for annotations. When <a href="https://www.scichart.com/documentation/js/v4/typedoc/interfaces/iannotation.html" rel="noopener noreferrer" target="_blank">IAnnotationðŸ“˜</a> is attached to <a href="https://www.scichart.com/documentation/js/v4/typedoc/classes/scichartsurface.html" rel="noopener noreferrer" target="_blank">SciChartSurfaceðŸ“˜</a> or to <a href="https://www.scichart.com/documentation/js/v4/typedoc/classes/scichartpolarsurface.html" rel="noopener noreferrer" target="_blank">SciChartPolarSurfaceðŸ“˜</a> a type check is performed, <a href="https://www.scichart.com/documentation/js/v4/typedoc/interfaces/iannotation.html#surfacetypes" rel="noopener noreferrer" target="_blank">annotation.surfaceTypesðŸ“˜</a> should contain the <a href="https://www.scichart.com/documentation/js/v4/typedoc/enums/esurfacetype.html" rel="noopener noreferrer" target="_blank">ESurfaceTypeðŸ“˜</a>. However, unlike the renderable series, the annotations can be compatible with normal surface type `ESurfaceType.SciChartSurfaceType`, polar surface type `ESurfaceType.SciChartPolarSurfaceType` or with both surface types at the same time.

In an annotation is only for polar surface it has "Polar" prefix. For example, [PolarPointerAnnotation](https://www.scichart.com/documentation/js/v4/2d-charts/annotations-api/polar-pointer-annotation). If annotation has no "Polar" prefix it can be either for normal surface only or for both depending on <a href="https://www.scichart.com/documentation/js/v4/typedoc/interfaces/iannotation.html#surfacetypes" rel="noopener noreferrer" target="_blank">annotation.surfaceTypesðŸ“˜</a> property.

**These are the annotations compatible with both surface types SciChartSurface and SciChartPolarSurface**:

- [CustomAnnotation](https://www.scichart.com/documentation/js/v4/2d-charts/annotations-api/custom-annotation)
- [LineAnnotation](https://www.scichart.com/documentation/js/v4/2d-charts/annotations-api/line-annotation)
- [TextAnnotation](https://www.scichart.com/documentation/js/v4/2d-charts/annotations-api/text-annotation)
