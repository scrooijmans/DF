# Class MouseManager

Go to [JavaScript Charting Documentation](https://www.scichart.com/documentation/js/v4/intro)

Go to <a href="https://scichart.com/demo" target="_blank">JavaScript Chart Examples</a>

Go to <a href="https://github.com/abtsoftware/scichart.js.examples" target="_blank">SciChart.js on Github</a>

The MouseManager handles mouse and touch events from any [IEventListenerSource](https://www.scichart.com/documentation/js/v4/typedoc/interfaces/ieventlistenersource.html) and publishes events to any [IReceiveMouseEvents](https://www.scichart.com/documentation/js/v4/typedoc/interfaces/ireceivemouseevents.html) type

remarks  
Used internally by the [SciChartSurface](https://www.scichart.com/documentation/js/v4/typedoc/classes/scichartsurface.html) and [SciChart3DSurface](https://www.scichart.com/documentation/js/v4/typedoc/classes/scichart3dsurface.html) to manage and route mouse eents

## Properties

### Protected doubleTapHandled

doubleTapHandled: boolean

Tracks status of double tap event. Positive value means that polyfill was used to handle the event.

### enableDoubleTapPolyfill

enableDoubleTapPolyfill: boolean = true

Toggles usage of double tap polyfill

### forceDoubleTapPolyfill

forceDoubleTapPolyfill: boolean = false

Defines whether double tap polyfill should always be used

### isOver

isOver: boolean = false

For subCharts - to track if the mousepointer is over the subChart, so we can fake Enter and Leave events

### Protected lastTapTime

lastTapTime: number

### maxTapDuration

maxTapDuration: number = 500

Max allowed interval between taps for them to be considered a double tap action.

remarks  
used by double tap polyfill
