SciChart.js is a cutting-edge next-generation JavaScript Chart Library. As a result, we are using the latest technologies to draw our fast, real-time and interactive JavaScript charts.

Minimum requirement for deployment of an application with SciChart.js is **your browser must support WebAssembly (wasm) and WebGL 1 (preferably WebGL 2).**

## FAQs<a href="https://www.scichart.com/documentation/js/v4/user-manual/minimum-browser-requirements/#faqs" class="hash-link" aria-label="Direct link to FAQs" translate="no" title="Direct link to FAQs">â€‹</a>

### Q: What ECMAScript (ES5, ES6) versions does SciChart.js Support?<a href="https://www.scichart.com/documentation/js/v4/user-manual/minimum-browser-requirements/#q-what-ecmascript-es5-es6-versions-does-scichartjs-support" class="hash-link" aria-label="Direct link to Q: What ECMAScript (ES5, ES6) versions does SciChart.js Support?" translate="no" title="Direct link to Q: What ECMAScript (ES5, ES6) versions does SciChart.js Support?">â€‹</a>

SciChart.js targets ES5 and supports JavaScript standard ECMAScript 5 (JavaScript 2009 edition). That means if your browser or a WebView support ES5 or higher, SciChart.js will run properly.

### Q: Do I need a GPU to run SciChart.js?<a href="https://www.scichart.com/documentation/js/v4/user-manual/minimum-browser-requirements/#q-do-i-need-a-gpu-to-run-scichartjs" class="hash-link" aria-label="Direct link to Q: Do I need a GPU to run SciChart.js?" translate="no" title="Direct link to Q: Do I need a GPU to run SciChart.js?">â€‹</a>

SciChart.js uses WebGL for rendering of 2D and 3D charts. You generally need a GPU to run WebGL, however specific requirements depend on the task and hardware. WebGL relies on the GPU for rendering, and most modern computers have integrated graphics like Intel Iris that are sufficient for most basic WebGL tasks, such as 2D Graphics or 3D scenes.

If the system lacks a GPU or has limited graphics capabilities, software rendering can still be used. For example, Google Chrome can use the SwiftShader software renderer, which emulates GPU functionality in software.

### Q: How can I check if my system supports WebGL?<a href="https://www.scichart.com/documentation/js/v4/user-manual/minimum-browser-requirements/#q-how-can-i-check-if-my-system-supports-webgl" class="hash-link" aria-label="Direct link to Q: How can I check if my system supports WebGL?" translate="no" title="Direct link to Q: How can I check if my system supports WebGL?">â€‹</a>

If your browser supports WebGL, then SciChart.js will run. You can check WebGL compatibility by visiting <a href="https://get.webgl.org/" rel="noopener noreferrer" target="_blank">https://get.webgl.org/</a> in your web browser. If your system supports WebGL, you will see a spinning cube. You can also check `chrome://gpu` in Chromium based browsers to see a report on whether WebGL is hardware accelerated, software emulated or not supported.

### Q: Does SciChart.js run on the Client or the Server?<a href="https://www.scichart.com/documentation/js/v4/user-manual/minimum-browser-requirements/#q-does-scichartjs-run-on-the-client-or-the-server" class="hash-link" aria-label="Direct link to Q: Does SciChart.js run on the Client or the Server?" translate="no" title="Direct link to Q: Does SciChart.js run on the Client or the Server?">â€‹</a>

At the moment, SciChart.js is a client-side library which renders on the client browser. Most clients machines (mobile devices, iPhone, Android phone, Raspberri Pi, Windows Desktop, macOS Desktop, Linux Desktop) will have at least a basic GPU and will support WebGL and therefore SciChart.js

### Q: Can SciChart.js run on ARM based embedded hardware or single-board computer (SBC) devices such as a Raspberry Pi?<a href="https://www.scichart.com/documentation/js/v4/user-manual/minimum-browser-requirements/#q-can-scichartjs-run-on-arm-based-embedded-hardware-or-single-board-computer-sbc-devices-such-as-a-raspberry-pi" class="hash-link" aria-label="Direct link to Q: Can SciChart.js run on ARM based embedded hardware or single-board computer (SBC) devices such as a Raspberry Pi?" translate="no" title="Direct link to Q: Can SciChart.js run on ARM based embedded hardware or single-board computer (SBC) devices such as a Raspberry Pi?">â€‹</a>

Yes, SciChart.js can run on any device which supports WebGL and WebAssembly, including ARM based devices such as the Raspberry Pi 4 and 5.

A list of Single-board computers (SBC's) which support WebGL can be found below:

- Raspberry Pi 4/5 - capable of OpenGL and WebGL with it's VideoCore VI GPU and updated Mesa drivers
- Orange Pi 5 - capable of OpenGL and WebGL due to the Rockchip RK3588(S) SoC and Mali-G610 GPU
- Asus Tinker Board - with a Rockchip RK3288 SoC offers strong graphics capabilities for WebGL
- Rock Pi 4 - With a Rockchip RK3399 SoC provides GPU capable of WebGL content
- Udoo Bolt - features an integrated GPU capable of OpenGL / WebGL
- ODROID N2+ - an industrial-grade SBC which offers robust performance for WebGL applications
- Libre Computer Project - support OpenGL ES 1.1/2.0 and WebGL however performance varies by model

![](out_scichartv4/user-manual/minimum-browser-requirements/index_media/f0173787a207cc0a720ca619ab3e16cfe886b3c7.svg)tip

To confirm an embedded device supports WebGL and hence SciChart.js, check the SBC's SoC (System on Chip) supports OpenGL, OpenGL ES and ensure a compatible Linux distribution with up to date Mesa drivers to run WebGL. WebGL support can then be checked in the browser by visiting `chrome://gpu` or <a href="https://get.webgl.org/" rel="noopener noreferrer" target="_blank">https://get.webgl.org/</a> in your browser.

See this blog post <a href="https://www.scichart.com/blog/how-to-run-raspberry-pi-os-on-vmware-player-view-realtime-javascript-charts-with-scichart-js/" rel="noopener noreferrer" target="_blank">how to setup Raspberry Pi OS on VMWare</a> which details how to ensure Mesa drivers are installed for the Chromium browser on a Linux subsystem by using Raspberry Pi OS and VMWare as an example.
