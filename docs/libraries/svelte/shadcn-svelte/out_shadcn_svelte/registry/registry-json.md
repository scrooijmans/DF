### aliases

`aliases` define how your registry's internal import paths will be transformed when users install your components. These should match how you import components within your registry code.

For example, if your registry's component has:

<figure data-rehype-pretty-code-figure="">
<pre class="no-scrollbar" data-language="svelte" data-theme="github-dark github-light-default" tabindex="0"><code>&lt;script lang=&quot;ts&quot;&gt;
  import { Button } from &quot;@/lib/registry/ui/button/index.js&quot;;
  import { cn } from &quot;@/lib/utils.js&quot;;
&lt;/script&gt;</code></pre>

</figure>

<figure data-metadata="" data-rehype-pretty-code-figure="">

  &quot;aliases&quot;: {
    &quot;lib&quot;: &quot;@/lib&quot;, // Matches your internal imports
    &quot;ui&quot;: &quot;@/lib/registry/ui&quot;, // Matches your internal imports

### overrideDependencies

`overrideDependencies` lets you force specific version ranges for dependencies, overriding what `shadcn-svelte registry build` detects in your `package.json`.

Common use cases:

- Using latest pre-release versions: `"overrideDependencies": ["paneforge@next"]`
- Pinning to specific versions: `"overrideDependencies": ["dep@1.5.0"]`

**Warning**: Overriding dependencies can lead to version conflicts if not carefully managed. This option should be used sparingly.

Example transformation:

<figure data-rehype-pretty-code-figure="">
<pre class="no-scrollbar" data-language="json" data-theme="github-dark github-light-default" tabindex="0"><code>// Your registry&#39;s package.json
{
  &quot;dependencies&quot;: {
    &quot;paneforge&quot;: &quot;1.0.0-next.1&quot;
  }
}</code></pre>
Copy <img src="out_shadcn_svelte/registry/registry-json/index_media/e1c2be166eeac758cef104bcc800e947e948608b.svg" class="lucide-icon lucide lucide-clipboard" />
</figure>

When the user installs your component, the latest `@next` version will be used instead of `1.0.0-next.1`

<figure data-rehype-pretty-code-figure="">
<pre class="no-scrollbar" data-language="json" data-theme="github-dark github-light-default" tabindex="0"><code>{
  &quot;dependencies&quot;: {
    &quot;paneforge&quot;: &quot;1.0.0-next.1&quot;, // overrideDependencies: []
    &quot;paneforge&quot;: &quot;1.0.0-next.5&quot; // overrideDependencies: [&quot;paneforge@next&quot;]
  }
}</code></pre>
