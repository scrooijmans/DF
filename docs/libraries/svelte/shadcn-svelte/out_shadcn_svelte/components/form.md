Component Source <img src="out_shadcn_svelte/components/form/index_media/10cce7474f3cd1bcf11a43a5ea07fe7e880c3138.svg" class="lucide-icon lucide lucide-code" />

Forms are tricky. They are one of the most common things you'll build in a web application, but also one of the most complex.

Well-designed HTML forms are:

- Well-structured and semantically correct.
- Easy to use and navigate (keyboard).
- Accessible with ARIA attributes and proper labels.
- Has support for client and server side validation.
- Well-styled and consistent with the rest of the application.

In this guide, we will take a look at building forms with <a href="https://formsnap.dev" class="font-medium underline underline-offset-4" rel="noopener noreferrer" target="_blank">formsnap</a>, <a href="https://superforms.rocks" class="font-medium underline underline-offset-4" rel="noopener noreferrer" target="_blank">sveltekit-superforms</a> and <a href="https://zod.dev" class="font-medium underline underline-offset-4" rel="noopener noreferrer" target="_blank">zod</a>.

## Features

The `Form` components offered by `shadcn-svelte` are wrappers around `formsnap` & `sveltekit-superforms` which provide a few things:

- Composable components for building forms.
- Form field components for scoping form state.
- Form validation using <a href="https://zod.dev" class="font-medium underline underline-offset-4" rel="noopener noreferrer" target="_blank">Zod</a> or any other validation library supported by <a href="https://superforms.rocks" class="font-medium underline underline-offset-4" rel="noopener noreferrer" target="_blank">Superforms</a>.
- Applies the correct `aria` attributes to form fields based on states.
- Enables you to easily use various components like <a href="https://www.shadcn-svelte.com/docs/components/select" class="font-medium underline underline-offset-4">Select</a>, <a href="https://www.shadcn-svelte.com/docs/components/radio-group" class="font-medium underline underline-offset-4">RadioGroup</a>, <a href="https://www.shadcn-svelte.com/docs/components/switch" class="font-medium underline underline-offset-4">Switch</a>, <a href="https://www.shadcn-svelte.com/docs/components/checkbox" class="font-medium underline underline-offset-4">Checkbox</a> and other form components with forms.

If you aren't familiar with <a href="https://superforms.rocks" class="font-medium underline underline-offset-4" rel="noopener noreferrer" target="_blank">Superforms</a> & <a href="https://formsnap.dev" class="font-medium underline underline-offset-4" rel="noopener noreferrer" target="_blank">Formsnap</a>, you should check out their documentation first, as this guide assumes you have a basic understanding of how they work together.

## Usage

### Create a form schema

Define the shape of your form using a Zod schema. You can read more about using Zod in the <a href="https://zod.dev" class="font-medium underline underline-offset-4" rel="noopener noreferrer" target="_blank">Zod documentation</a>. We're going to define it in a file called `schema.ts` in the same directory as our page component, but you can put it anywhere you like.

<pre class="no-scrollbar" data-language="ts" data-metadata="" data-theme="github-dark github-light-default" tabindex="0"><code>import { z } from &quot;zod&quot;;
 
export const formSchema = z.object({
  username: z.string().min(2).max(50),
});
 
export type FormSchema = typeof formSchema;</code></pre>

### Create form component

For this example, we'll be passing the `form` returned from the load function as a prop to this component. To ensure it's typed properly, we'll use the `SuperValidated` type from `sveltekit-superforms`, and pass in the type of our form schema.

<figure data-metadata="" data-rehype-pretty-code-figure="">
<pre class="no-scrollbar" data-language="svelte" data-metadata="" data-theme="github-dark github-light-default" tabindex="0"><code>&lt;script lang=&quot;ts&quot;&gt;
  import * as Form from &quot;$lib/components/ui/form/index.js&quot;;
  import { Input } from &quot;$lib/components/ui/input/index.js&quot;;
  import { formSchema, type FormSchema } from &quot;./schema&quot;;
  import {
    type SuperValidated,
    type Infer,
    superForm,
  } from &quot;sveltekit-superforms&quot;;
  import { zod4Client } from &quot;sveltekit-superforms/adapters&quot;;
 
  let { data }: { data: { form: SuperValidated&lt;Infer&lt;FormSchema&gt;&gt; } } =
    $props();
 
  const form = superForm(data.form, {
    validators: zod4Client(formSchema),
  });
 
  const { form: formData, enhance } = form;
&lt;/script&gt;
 
&lt;form method=&quot;POST&quot; use:enhance&gt;

      {#snippet children({ props })}
        &lt;Form.Label&gt;Username&lt;/Form.Label&gt;
        &lt;Input {...props} bind:value={$formData.username} /&gt;
      {/snippet}
    &lt;/Form.Control&gt;
    &lt;Form.Description&gt;This is your public display name.&lt;/Form.Description&gt;

&lt;/form&gt;</code></pre>

### Create an Action

<figure data-metadata="" data-rehype-pretty-code-figure="">
<pre class="no-scrollbar" data-language="ts" data-metadata="" data-theme="github-dark github-light-default" tabindex="0"><code>import type { PageServerLoad, Actions } from &quot;./$types.js&quot;;
import { fail } from &quot;@sveltejs/kit&quot;;
import { superValidate } from &quot;sveltekit-superforms&quot;;
import { zod4 } from &quot;sveltekit-superforms/adapters&quot;;
import { formSchema } from &quot;./schema&quot;;
 
export const load: PageServerLoad = async () =&gt; {
  return {
    form: await superValidate(zod4(formSchema)),
  };
};
 
export const actions: Actions = {
  default: async (event) =&gt; {
    const form = await superValidate(event, zod(formSchema));
    if (!form.valid) {
      return fail(400, {
        form,
      });
    }
    return {
      form,
    };
  },
};</code></pre>
