# \<MiniMap /\>

<a href="https://github.com/xyflow/xyflow/blob/main/packages/svelte/src/lib/plugins/Minimap/Minimap.svelte" class="x:focus-visible:nextra-focus x:text-primary-600 x:underline x:hover:no-underline x:decoration-from-font x:[text-underline-position:from-font]" rel="noreferrer" target="_blank">Source on GitHub <img src="out_svelteflow/api-reference/components/mini-map/index_media/926b7c4e31213c3a8240847d8251d75a921b6a21.svg" class="x:inline x:align-baseline x:shrink-0" /></a>

The `<MiniMap />` component can be used to render an overview of your flow. It renders each node as an SVG element and visualizes where the current viewport is in relation to the rest of the flow.

``` x:group
<script lang="ts">
  import { SvelteFlow, MiniMap } from '@xyflow/svelte';
  import '@xyflow/svelte/dist/style.css';
 
  let nodes = $state.raw([]);
  let edges = $state.raw([]);
</script>
 
<SvelteFlow bind:nodes bind:edges>
  <MiniMap nodeStrokeWidth={3} />
</SvelteFlow>
```

The type for props of `<MiniMap />` component is exported as `MiniMapProps`. Additionally, it extends the props of `<div />`.

## Examples<a href="https://svelteflow.dev/api-reference/components/mini-map#examples" class="x:focus-visible:nextra-focus subheading-anchor" aria-label="Permalink for this section"></a>

### Making the mini map interactive<a href="https://svelteflow.dev/api-reference/components/mini-map#making-the-mini-map-interactive" class="x:focus-visible:nextra-focus subheading-anchor" aria-label="Permalink for this section"></a>

By default, the mini map is non-interactive. To allow users to interact with the viewport by panning or zooming the minimap, you can set either of the `zoomable` or `pannable` (or both!) props to `true`.

``` x:group
<script lang="ts">
  import { SvelteFlow, MiniMap } from '@xyflow/svelte';
  import '@xyflow/svelte/dist/style.css';
 
  let nodes = $state.raw([]);
  let edges = $state.raw([]);
</script>
 
 
<SvelteFlow bind:nodes bind:edges>
  <MiniMap pannable zoomable />
</SvelteFlow>
```

### Customising mini map node color<a href="https://svelteflow.dev/api-reference/components/mini-map#customising-mini-map-node-color" class="x:focus-visible:nextra-focus subheading-anchor" aria-label="Permalink for this section"></a>

The `nodeColor`, `nodeStrokeColor`, and `nodeClassName` props can be a function that takes a <a href="https://svelteflow.dev/api-reference/types/node" class="x:focus-visible:nextra-focus x:text-primary-600 x:underline x:hover:no-underline x:decoration-from-font x:[text-underline-position:from-font]"><code class="nextra-code" dir="ltr">Node</code></a> and computes a value for the prop. This can be used to customize the appearance of each mini map node.

This example shows how to color each mini map node based on the node’s type:

``` x:group
<script lang="ts">
  import { SvelteFlow, MiniMap } from '@xyflow/svelte';
  import '@xyflow/svelte/dist/style.css';
 
  let nodes = $state.raw([]);
  let edges = $state.raw([]);
 
  function nodeColor(node) {
    return node.type === 'input' ? 'blue' : 'red';
  }
</script>
 
<SvelteFlow bind:nodes bind:edges>
  <MiniMap nodeColor={nodeColor} />
</SvelteFlow>
```
