## Pagination & Child Rows <a href="https://www.ag-grid.com/javascript-data-grid/row-pagination/#pagination--child-rows" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/row-pagination/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

Both [Row Grouping](https://www.ag-grid.com/javascript-data-grid/grouping/) and [Master Detail](https://www.ag-grid.com/javascript-data-grid/master-detail/) have rows that expand. When this happens, consideration needs to be given as to how this impacts the number of rows on the page. There are two modes of operation that can be used depending on what your application requirements.

### Mode 1: Paginate Only Top Level Rows <a href="https://www.ag-grid.com/javascript-data-grid/row-pagination/#mode-1-paginate-only-top-level-rows" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/row-pagination/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

The first mode is the default. The rows are split according to the top level rows. For example if row grouping with a page size of 10, then each page will contain 10 top level groups. When expanding a group with this mode, all children for that group, along with the 10 original groups for that page, will get displayed in one page. This will result in a page size greater than the initial page size of 10 rows.

This mode is typically best suited for Row Grouping as children are always displayed alongside the parent group. It is also typically best for Master Detail, as detail rows (that typically contain detail tables) will always appear below their master rows.

In the example below, note the following:

- Each page will always contain exactly 10 groups.
- Expanding a group will not push rows to the next page.

<img src="out_ag_grid/row-pagination/index_media/8ded47c1186e17f7830cf2d4518c3630384cfc60.svg" class="logomark" />

### Mode 2: Paginate All Rows, Including Children <a href="https://www.ag-grid.com/javascript-data-grid/row-pagination/#mode-2-paginate-all-rows-including-children" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/row-pagination/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

The second mode paginates all rows, including child rows when Row Grouping and detail rows with Master Detail. For example if row grouping with a page size of 10, then each page will always contain exactly 10 rows, even if it means having children appear on a page after the page containing the parent. This can be particularly confusing if the last row of a page is expanded, as the children will appear on the next page (not visible to the user unless they navigate to the next page).

This modes is typically best if the application never wants to exceed the maximum number of rows in a page past the page size. This can be helpful if designing for touch devices (e.g. tablets) where UX requirements state no scrolls should be visible in the application - paging to a strict page size can guarantee no vertical scrolls will appear.

To enable pagination on all rows, including children, set grid property `paginateChildRows=true`.

In the example below, note the following:

- Each page will always contain exactly 10 rows (not groups).
- Expanding a group will push rows to the next page to limit the total number of rows to 10.

<img src="out_ag_grid/row-pagination/index_media/ae2971b6b482d433157d4aac53e206fe28cf84e4.svg" class="_icon_1jqgc_1 icon" />

When `paginateChildRows=true` the Grid automatically disables Group Rows Sticky, see: [suppressGroupRowsSticky](https://www.ag-grid.com/javascript-data-grid/grid-options/#reference-rowGrouping-suppressGroupRowsSticky).

<img src="out_ag_grid/row-pagination/index_media/8ded47c1186e17f7830cf2d4518c3630384cfc60.svg" class="logomark" />

### Fallback to Mode 2 <a href="https://www.ag-grid.com/javascript-data-grid/row-pagination/#fallback-to-mode-2" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/row-pagination/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

If using either of the following features, the grid will be forced to use the second mode:

- [Hide Open Parents](https://www.ag-grid.com/javascript-data-grid/grouping-multiple-group-columns/#hiding-expanded-parent-rows)
- [Hiding Parents of Individual Rows](https://www.ag-grid.com/javascript-data-grid/grouping-data/#hiding-parents-of-individual-rows)

This is because both of these features remove top level rows (group rows and master rows) from the displayed rows, making it impossible to paginate based on the top level rows only.
