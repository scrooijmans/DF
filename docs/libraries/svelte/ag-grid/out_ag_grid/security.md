<span class="image placeholder _frameworkLogo_192zo_1" original-image-src="https://www.ag-grid.com/_astro/javascript.DOqolN0O.svg" original-image-title="">javascript logo</span>JavaScript<img src="out_ag_grid/security/index_media/ebb52304aa2fe75a94cd1555a353273f67162700.svg" class="lucide lucide-chevron-down _chevronDown_6498a_108" />

The grid allows you to work with security tools and parameters to make your application meet your business requirements.

## Content Security Policy (CSP) <a href="https://www.ag-grid.com/javascript-data-grid/security/#content-security-policy-csp" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/security/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

The basic information on Content Security Policy can be found on the [MDN web docs](https://developer.mozilla.org/en-US/docs/Web/HTTP/CSP) website and will cover the necessary information on the subject. The grid works with CSP, but some basic configuration is necessary to have your application load correctly. Below is detailed what the minimum set of CSP rules for the grid is and why.

### script-src <a href="https://www.ag-grid.com/javascript-data-grid/security/#script-src" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/security/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

Some optional grid features compile string expressions into code, and if you're using these then the `script-src 'unsafe-eval'` directive is required. However, it is possible to avoid this requirement as listed below:

- Many grid properties such as [Cell Class Rules](https://www.ag-grid.com/javascript-data-grid/cell-styles/#cell-class-rules) and [Value Getters](https://www.ag-grid.com/javascript-data-grid/value-getters/) allow using expressions. However, this requires `script-src 'unsafe-eval'`. In order to avoid this requirement, replace expressions with functions as shown in the example below for the cellClassRules property of a column definition:
  - `{ 'is-negative': 'value < 0' }` requires `script-src 'unsafe-eval'` so that `'value < 0'` can be compiled into a function.
  - `{ 'is-negative': params => params.value < 0 }` does not because you have provided a function.

### style-src <a href="https://www.ag-grid.com/javascript-data-grid/security/#style-src" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/security/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

[Themes](https://www.ag-grid.com/javascript-data-grid/theming/) work by injecting CSS styles into the DOM when the grid initialises. By default this requires a `style-src 'unsafe-inline'` directive in your CSP.

However, you can avoid this requirement using the `styleNonce` grid option as shown below. The `styleNonce` property should be set to a random value that changes with each page load.

``` code
const gridOptions = {
    styleNonce: "416d1177",

    // other grid options ...
}
```

This sets the nonce attribute on the style tag, so the grid will now work with the CSP directive `style-src 'nonce-416d1177'` without the need for `'unsafe-inline'`.

CSP nonces are global to a page, so when a page has multiple grids, all grids must have the same styleNonce set. Consider using [global grid options](https://www.ag-grid.com/javascript-data-grid/grid-interface/#global-grid-options) to set the styleNonce property to all grids.

### img-src <a href="https://www.ag-grid.com/javascript-data-grid/security/#img-src" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/security/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

The `img-src data:` directive is required because the grid uses data urls to embed SVG images in CSS files.

### font-src <a href="https://www.ag-grid.com/javascript-data-grid/security/#font-src" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/security/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

If you are using [legacy themes](https://www.ag-grid.com/javascript-data-grid/theming-v32/) then the `font-src data:` directive is required. This is because legacy themes use data urls to embed the icon font CSS files. Theming API themes use SVG icons and so do not require this.

If you use the `loadThemeGoogleFonts` grid option, fonts will be loaded from the Google font CDN, which requires the `font-src fonts.gstatic.com` directive.

### Example locked-down CSP <a href="https://www.ag-grid.com/javascript-data-grid/security/#example-locked-down-csp" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/security/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

This example CSP below allows the grid to be run in a secure manner:

``` code
<meta http-equiv="Content-Security-Policy"
      content="default-src 'self'; style-src 'self' 'nonce-123xyz'; img-src 'self' data:">
```

It assumes:

1.  The application loads assets from the same server (`'self'`)
2.  You do not use string expressions in grid options
3.  You are using [Theming API](https://www.ag-grid.com/javascript-data-grid/theming/) instead of [legacy themes](https://www.ag-grid.com/javascript-data-grid/theming-v32/), and are not using loadThemeGoogleFonts
4.  The random string passed to `gridOptions.styleNonce` on this page load is "123xyz"
