## Syncing Detail Scrolling with Master <a href="https://www.ag-grid.com/javascript-data-grid/master-detail-other/#syncing-detail-scrolling-with-master" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/master-detail-other/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

By default, the Detail Grid takes up the width of the Master Grid and does not move when the Master Grid's columns are horizontally scrolled. This is because the Detail Grid is not sitting with the other Master Grid's cells, rather it is in a separate container that overlays the Master Grid's cells and takes up the full width of the grid ignoring all the Master Grid's columns.

The underlying feature of the grid that allows the Detail Grid to span the width of the Master Grid is called [Full Width Row](https://www.ag-grid.com/javascript-data-grid/full-width-rows/).

It is possible to have the Detail Grid sit within the same container as the Master Grid's cells and hence move with the Master Grid's horizontal scrolling. This is achieved by [Embedding the Full Width Row](https://www.ag-grid.com/javascript-data-grid/full-width-rows/#embedded-full-width-rows) and is set via the grid property `embedFullWidthRows=true` for the Master Grid. This tells the grid to layout (embed) the Detail Panel with the other rows.

In the example below, notice that horizontal scrolling is combined for both Master Grid and Detail Grid using the Embed Full Width Rows feature.

<img src="out_ag_grid/master-detail-other/index_media/8ded47c1186e17f7830cf2d4518c3630384cfc60.svg" class="logomark" />

If you are mixing Embed Full Width Rows and [Custom Detail](https://www.ag-grid.com/javascript-data-grid/master-detail-custom-detail/), then be aware the Detail Panel will get rendered three times as follows:

- Pinned Left Columns
- Pinned Right Columns
- Pinned Centre Columns

This is because the Columns, and thus the Detail Panel, are appearing in three separate scrollable sections. The example below demonstrates this. Note the custom Detail Panel appears in all three sections.

<img src="out_ag_grid/master-detail-other/index_media/8ded47c1186e17f7830cf2d4518c3630384cfc60.svg" class="logomark" />

If you want your custom Detail Panel to only show content in one section, have logic that inspects the components `params.pinned` property and renders content relevant for the section.

## Tree Data <a href="https://www.ag-grid.com/javascript-data-grid/master-detail-other/#tree-data" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/master-detail-other/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

The Master / Detail is supported with [Tree Data](https://www.ag-grid.com/javascript-data-grid/tree-data/). Both leaf nodes and groups can be master rows. If a group is also a master row, the detail grid will be shown before its children.

<img src="out_ag_grid/master-detail-other/index_media/ae2971b6b482d433157d4aac53e206fe28cf84e4.svg" class="_icon_1jqgc_1 icon" />

Master / Detail will by default show detail grids for every row, including groups. To show detail grids for leaf nodes only, provide the `isRowMaster` callback and return `true` for leaf nodes only.

When using [Tree Data Paths](https://www.ag-grid.com/javascript-data-grid/tree-data-paths/), filler groups cannot be master rows and `isRowMaster` will not be called.

<img src="out_ag_grid/master-detail-other/index_media/8ded47c1186e17f7830cf2d4518c3630384cfc60.svg" class="logomark" />

### Layouts <a href="https://www.ag-grid.com/javascript-data-grid/master-detail-other/#layouts" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/master-detail-other/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

It is not possible to mix [DOM layout](https://www.ag-grid.com/javascript-data-grid/grid-size/#dom-layout) for master detail. This is because the layout is a CSS setting that would be inherited by all grids contained with the master grid. So if your master grid was 'for-print', then all child grids would pick up the 'for-print' layout.

When using Master / Detail and [for-print](https://www.ag-grid.com/javascript-data-grid/printing/), then all detail grids need to use for-print.

When using Master / Detail and [auto-height](https://www.ag-grid.com/javascript-data-grid/grid-size/#auto-height-layout), then all detail grids need to use auto-height.

### Cell Selection <a href="https://www.ag-grid.com/javascript-data-grid/master-detail-other/#cell-selection" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/master-detail-other/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

When [Cell Selection](https://www.ag-grid.com/javascript-data-grid/cell-selection/) is enabled on the Master Grid, the Detail Grid will not participate in the Cell Selection of the Master Grid.
