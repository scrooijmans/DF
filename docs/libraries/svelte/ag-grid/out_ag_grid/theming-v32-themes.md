## Applying a v32 Theme to an App <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#applying-a-v32-theme-to-an-app" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

To use a theme, add the theme class name to the `div` element that contains your grid. The following is an example of using the Quartz theme:

``` code
<div id="myGrid" class="ag-theme-quartz"></div>
```

<img src="out_ag_grid/theming-v32-themes/index_media/c72ab96ed63a5cb43751440572e61cb5c424d4f0.svg" class="_icon_1jqgc_1 icon" />

The grid must always have a theme class set on its container, whether this is a provided theme or your own.

## Loading CSS files and fonts <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#loading-css-files-and-fonts" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

In order for the above code to work, the correct stylesheets must be loaded in the correct order. There are two kinds of stylesheet that need to be loaded when using provided themes:

- **`ag-grid.css`** - structural styles containing CSS rules that are essential to the functioning of the grid and must be loaded first.
- **`ag-theme-{theme-name}.css`** - theme styles that add design look and feel on top of the structural styles.

<img src="out_ag_grid/theming-v32-themes/index_media/ae2971b6b482d433157d4aac53e206fe28cf84e4.svg" class="_icon_1jqgc_1 icon" />

The correct files to load are located in `ag-grid-community/styles`.

This path has changed in v28, and the old files are still there as part of the [Legacy Styles](https://www.ag-grid.com/javascript-data-grid/theming-v32-upgrading-to-v28/) but will be removed in v29.

Double-check that you are importing files from the new paths. If you have `/src/` or `/dist/` in your path then you're using the old paths.

There are various ways to load these stylesheets, as described in the sections below:

### Pre-built Bundles <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#pre-built-bundles" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

Some pre-built bundles, whether downloaded from our website or included in the `ag-grid-community` [NPM package](https://www.ag-grid.com/javascript-data-grid/installation/#npm-installation), already embed the structural styles and all provided themes. If you are using one of these files, you do not need to load separately CSS.

### Loading 3rd party fonts <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#loading-3rd-party-fonts" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

Some themes have a recommended 3rd party font, which is not bundled with the theme CSS. If the font is not available at runtime, the OS default UI font will be used instead - Segoe UI on Windows and SF Pro on Mac.

#### Loading IBM Plex Sans for the Quartz theme <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#loading-ibm-plex-sans-for-the-quartz-theme" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

Plex Sans is a font that gives excellent readability on both regular and high DPI screens. It features a tall x-height to aid in readability of mixed-case and lower-case text.

You can [download Plex Sans from Google Fonts](https://fonts.google.com/specimen/IBM+Plex+Sans) (you need weights 400/Regular and 500/Medium)and use it in your application, or load it from the Google Fonts API:

``` code
<!-- in HTML -->
<link href="https://fonts.googleapis.com/css2?family=IBM+Plex+Sans:wght@400;500;700&display=swap" rel="stylesheet" />
```

``` code
/* in css */
@import "https://fonts.googleapis.com/css2?family=IBM+Plex+Sans:wght@400;500;700&display=swap";
```

#### Loading Roboto for the Material theme <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#loading-roboto-for-the-material-theme" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

Roboto is the font recommended by the Material Design specification.

You can [download Roboto from Google Fonts](https://fonts.google.com/specimen/Roboto) and use it in your application, or load it from the Google Fonts API:

``` code
<!-- in HTML -->
<link href="https://fonts.googleapis.com/css?family=Roboto" rel="stylesheet" />
```

``` code
/* is css */
@import "https://fonts.googleapis.com/css?family=Roboto";
```

### JavaScript Bundlers <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#javascript-bundlers" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

If you are using a JavaScript bundler like webpack or Rollup and it is configured to load styles, you can `require()` the correct CSS file from `node_modules`. This is the recommended approach as webpack will take care of minifying your CSS in production:

``` code
// CommonJS:
require("ag-grid-community/styles/ag-grid.css");

// ECMAScript Modules:
import "ag-grid-community/styles/ag-grid.css";
```

### App Hosted <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#app-hosted" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

You can copy, either manually or as part of your app's build, the required CSS files (`ag-grid.css` and `ag-theme-{theme-name}.css`) from node_modules and serve it with your app.

### Sass Styling API <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#sass-styling-api" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

If you're using the [Sass Styling API](https://www.ag-grid.com/javascript-data-grid/theming-v32-customisation-sass/) then the right CSS files will be automatically included for your chosen theme. For projects that are already using Sass this is the recommended approach.

### CDN <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#cdn" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

You can load the structural styles and theme from a free CDN by adding this code to your page.

``` code
<link
        rel="stylesheet"
        href="https://cdn.jsdelivr.net/npm/ag-grid-community@34.2.0/styles/ag-grid.css" />

<link
        rel="stylesheet"
        href="https://cdn.jsdelivr.net/npm/ag-grid-community@34.2.0/styles/ag-theme-quartz.css" />
```

<img src="out_ag_grid/theming-v32-themes/index_media/ae2971b6b482d433157d4aac53e206fe28cf84e4.svg" class="_icon_1jqgc_1 icon" />

Change the theme name in the URL to the one that you're using, and ensure that the version number in the URL matches the JS version you're using. This is useful for testing and prototyping but not recommended for production as your app will be unavailable if the jsdelivr servers are down.

## Creating your own theme <a href="https://www.ag-grid.com/javascript-data-grid/theming-v32-themes/#creating-your-own-theme" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/theming-v32-themes/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

The majority of users select a provided theme and [make customisations using CSS](https://www.ag-grid.com/javascript-data-grid/theming-v32-customisation/). If your chosen provided theme has elements that you don't want, you will need to add CSS rules to remove them. If your desired look and feel is very different from the provided theme, at some point it becomes easier to start from scratch. To do this, you can define your own theme.

A theme is simply a CSS class name matching the pattern `ag-theme-*`, along with CSS rules that target this class name.

<img src="out_ag_grid/theming-v32-themes/index_media/c72ab96ed63a5cb43751440572e61cb5c424d4f0.svg" class="_icon_1jqgc_1 icon" />

If you create your own theme, its name must start with `ag-theme-`.

Ensure that `ag-grid.css` is loaded, choose a theme name and apply it to the grid:

``` code
<div id="myGrid" class="ag-theme-mycustomtheme"></div>
```

That's it! You've created a theme. You haven't added any styles to it so what you will see is a nearly blank slate - basic customisable borders and padding but no opinionated design elements. You can then [add customisations using CSS](https://www.ag-grid.com/javascript-data-grid/theming-v32-customisation/):

``` code
.ag-theme-mycustomtheme {
    /* customise with CSS variables */
    --ag-grid-size: 8px;
    --ag-border-color: red;
}
.ag-theme-mycustomtheme .ag-header {
    /* or with CSS selectors targeting grid DOM elements */
    font-style: italic;
}
```
