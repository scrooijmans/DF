# JavaScript Data GridRow Overview

<span class="image placeholder _frameworkLogo_192zo_1" original-image-src="https://www.ag-grid.com/_astro/javascript.DOqolN0O.svg" original-image-title="">javascript logo</span>JavaScript<img src="out_ag_grid/row-interface/index_media/ebb52304aa2fe75a94cd1555a353273f67162700.svg" class="lucide lucide-chevron-down _chevronDown_6498a_108" />

Every row displayed in the grid is represented by a Row Node which exposes stateful attributes and methods for directly interacting with the row.

## Row Node <a href="https://www.ag-grid.com/javascript-data-grid/row-interface/#row-node" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/row-interface/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

Row Nodes implement the `IRowNode<TData>` interface and in most cases wrap individual data items as supplied to the grid. Row nodes also contain runtime information about the row such as its current rowIndex. The Row Node contains attributes, methods and emits events. Additional attributes are used if the Row Node is a group.

See [Row Reference](https://www.ag-grid.com/javascript-data-grid/row-object/) for a complete list of attributes / methods associated with rows.

## Row Events <a href="https://www.ag-grid.com/javascript-data-grid/row-interface/#row-events" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/row-interface/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

Row Nodes fire events as they are updated which makes it possible to trigger logic in your application.

<img src="out_ag_grid/row-interface/index_media/ae2971b6b482d433157d4aac53e206fe28cf84e4.svg" class="_icon_1jqgc_1 icon" />

Most users will not need to use Row Events but they are made available for advanced use cases. Please consider the guidance below.

### Row Event Guidance <a href="https://www.ag-grid.com/javascript-data-grid/row-interface/#row-event-guidance" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/row-interface/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

All events fired by the `rowNode` are synchronous (events are normally asynchronous). The grid is also listening for these events internally. This means that when you receive an event, the grid may still have some work to do (e.g. if `rowTop` changed, the grid UI may still have to update to reflect this change).

It is recommended to **not** call any grid API functions while receiving events from the `rowNode`. Instead, it is best to put your logic into a timeout and call the grid in another VM tick.

### Row Event Listener Lifecycle <a href="https://www.ag-grid.com/javascript-data-grid/row-interface/#row-event-listener-lifecycle" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/row-interface/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

When adding event listeners to a row, they will stay with the row until the row is destroyed. For example, if you set new data into the grid, all listeners on the old data will be removed.

#### Row Events in Cell Components <a href="https://www.ag-grid.com/javascript-data-grid/row-interface/#row-events-in-cell-components" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/row-interface/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

To avoid memory leaks, when adding listeners to `rowNodes` in cell renderers, ensure that you remove the listener when the cell renderer is destroyed. You can cater for this as follows:

``` code
class MyCellRenderer {

    private selectionChangedCallback;
    private params: ICellRendererParams;

    init(params: ICellRendererParams) {
        this.params = params;
        // add listener
        this.selectionChangedCallback = () => {
            console.log('node selected = ' + params.node.isSelected());
        };

        params.node.addEventListener('rowSelected', this.selectionChangedCallback);
    }

    // OTHER METHODS...

    destroy() {
        // remove listener on destroy
        this.params.node.removeEventListener('rowSelected', this.selectionChangedCallback);
    }
}
```

See [Row Events](https://www.ag-grid.com/javascript-data-grid/row-events/) for a complete list of row events.
