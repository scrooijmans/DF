# JavaScript Data GridSSRM Pagination

<span class="image placeholder _frameworkLogo_192zo_1" original-image-src="https://www.ag-grid.com/_astro/javascript.DOqolN0O.svg" original-image-title="">javascript logo</span>JavaScript<img src="out_ag_grid/server-side-model-pagination/index_media/ebb52304aa2fe75a94cd1555a353273f67162700.svg" class="lucide lucide-chevron-down _chevronDown_6498a_108" />

Enterprise<img src="out_ag_grid/server-side-model-pagination/index_media/7e74cacfc7ea261f2c66286804408c45e78f84a2.svg" class="_icon_1jqgc_1 icon" />

If you are dealing with large amounts of data, your applications may decide to use pagination to help the user navigate through the data.

## Enabling Pagination <a href="https://www.ag-grid.com/javascript-data-grid/server-side-model-pagination/#enabling-pagination" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/server-side-model-pagination/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

Pagination is enabled in the grid via the `pagination` grid option. The pagination page size is typically set alongside this using the `paginationPageSize` option. These options are shown below:

``` code
const gridOptions = {
    // enables pagination in the grid
    pagination: true,

    // sets 10 rows per page (default is 100)
    paginationPageSize: 10,

    // allows the user to select the page size from a predefined list of page sizes
    paginationPageSizeSelector: [10, 20, 50, 100],

    // other grid options ...
}
```

For more configuration details see the section on [Pagination](https://www.ag-grid.com/javascript-data-grid/row-pagination/).

## Server-Side Pagination <a href="https://www.ag-grid.com/javascript-data-grid/server-side-model-pagination/#server-side-pagination" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/server-side-model-pagination/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

The actual pagination of rows is performed on the server when using the Server-Side Row Model. When the grid needs more rows it makes a request via `getRows(params)` on the [Server-Side Datasource](https://www.ag-grid.com/javascript-data-grid/server-side-model-datasource/) with metadata containing pagination details.

The properties relevant to pagination in the request are shown below:

``` code
// IServerSideGetRowsRequest
{
    // first row requested
    startRow: number,

    // index after last row requested
    endRow: number,

... // other params
}
```

The `endRow` requested by the grid may not actually exist in the data so the correct `lastRowIndex` should be supplied in the response to the grid. See [Server-Side Datasource](https://www.ag-grid.com/javascript-data-grid/server-side-model-datasource/) for more details.

## Example: Server-Side Pagination <a href="https://www.ag-grid.com/javascript-data-grid/server-side-model-pagination/#example-server-side-pagination" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/server-side-model-pagination/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

The example below demonstrates server-side Pagination. Note the following:

- Pagination is enabled using the grid option `pagination=true`.
- A pagination page size of 10 (default is 100) is set using the grid option `paginationPageSize=10`.
- The number of rows returned per request is set to 10 (default is 100) using `cacheBlockSize=10`.
- Use the arrows in the pagination panel to traverse the data. Note the last page arrow is greyed out as the last row index is only supplied to the grid when the last row has been reached.
- Open the browser's dev console to view the request supplied to the datasource.

## Pagination with Groups <a href="https://www.ag-grid.com/javascript-data-grid/server-side-model-pagination/#pagination-with-groups" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/server-side-model-pagination/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

When grouping, pagination splits rows according to top-level groups only. This has the following implications:

- The number of pages is determined by the number of top-level rows and not children
- When groups are expanded, the number of pagination pages does not change.
- When groups are expanded, all children rows appear on the same page as the parent row.

The example below demonstrates pagination with grouping. Note the following:

- No block size is specified so 100 rows per block is used.
- Grid property `paginationAutoPageSize=true` is set. This means the number of displayed rows is automatically set to the number of rows that fit the vertical scroll, so no vertical scroll is present.
- As rows are expanded, the number of visible rows in a page grows. The children appear on the same row as the parent and no rows are pushed to the next page.
- For example, expand 'Australia' which will result in a large list for which vertical scrolling will be needed to view all children.

<img src="out_ag_grid/server-side-model-pagination/index_media/8ded47c1186e17f7830cf2d4518c3630384cfc60.svg" class="logomark" />

## Pagination with Child Rows <a href="https://www.ag-grid.com/javascript-data-grid/server-side-model-pagination/#pagination-with-child-rows" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/server-side-model-pagination/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

If it is desired to keep the row count exactly at the page size, then set grid property `paginateChildRows=true`.

This will have the effect that child rows will get included in the pagination calculation. This will mean if a group is expanded, the pagination will split the child rows across pages and also possibly push later groups into later pages.

The example below demonstrates pagination with grouping and `paginateChildRows=true`. Note the following:

- No block size is specified thus 100 rows per block is used.

- Grid property `paginationAutoPageSize=true` is set. This means the number of displayed rows is automatically set to the number of rows that fit the vertical scroll.

- As rows are expanded, the number of visible rows in each page is fixed. This means expanding groups will push rows to the next page. This includes later group rows and also its own child rows (if the child rows don't fit on the current page).

- If the last visible row is expanded, the grid gives a confusing user experience, as the rows appear on the next page. So the user will have to click 'expand' and then click 'next page' to see the child rows. This is the desired behaviour as the grid keeps the number of rows on one page consistent. If this behaviour is not desired, then do not use `paginationAutoPageSize=true`.
