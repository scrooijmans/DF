# JavaScript Data GridSSRM Master Detail

<span class="image placeholder _frameworkLogo_192zo_1" original-image-src="https://www.ag-grid.com/_astro/javascript.DOqolN0O.svg" original-image-title="">javascript logo</span>JavaScript<img src="out_ag_grid/server-side-model-master-detail/index_media/ebb52304aa2fe75a94cd1555a353273f67162700.svg" class="lucide lucide-chevron-down _chevronDown_6498a_108" />

Enterprise<img src="out_ag_grid/server-side-model-master-detail/index_media/7e74cacfc7ea261f2c66286804408c45e78f84a2.svg" class="_icon_1jqgc_1 icon" />

This section shows how the Server-Side Row Model can be configured with a Master / Detail view.

The ability to nest grids within grids is commonly referred to as Master / Detail. Here the top-level grid is referred to as the 'master grid' and the nested grid is referred to as the 'detail grid'.

Master / Details is configured the same way for the Server-Side Row Model and the Client-Side Row Model. For a comprehensive look at Master / Detail configurations, see: [Client-Side Master / Detail](https://www.ag-grid.com/javascript-data-grid/master-detail/).

Because the configuration is already discussed in [Client-Side Master / Detail](https://www.ag-grid.com/javascript-data-grid/master-detail/), this page focuses on areas that are of particular interest to this Server-Side version.

## Enabling Master / Detail <a href="https://www.ag-grid.com/javascript-data-grid/server-side-model-master-detail/#enabling-master--detail" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/server-side-model-master-detail/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

To enable Master / Detail, you should set the following grid options:

- **masterDetail:** Set to `true` to inform the grid you want to allow expanding of rows to reveal detail grids.
- **detailGridOptions:** The grid options to set for the detail grid. The detail grid is a fully featured instance of AG Grid, so any configuration can be set on it that you would set on any other grid.
- **getDetailRowData:** A function you implement to provide the grid with rows for display in the detail grids.

These grid options are illustrated below:

``` code
const gridOptions = {
    // master grid columns
    columnDefs: [],

    // use the server-side row model
    rowModelType: 'serverSide',

    // enable master detail
    masterDetail: true,

    detailCellRendererParams: {
        detailGridOptions: {
            // detail grid columns
            columnDefs: [],
        },
        getDetailRowData: params => {
            // supply data to the detail grid
            params.successCallback(params.data);
        }
    },

    // other grid options ...
}
```

<img src="out_ag_grid/server-side-model-master-detail/index_media/ae2971b6b482d433157d4aac53e206fe28cf84e4.svg" class="_icon_1jqgc_1 icon" />

Note that the nested detail grid can be configured to use any Row Model.

### Expanding Master Rows <a href="https://www.ag-grid.com/javascript-data-grid/server-side-model-master-detail/#expanding-master-rows" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/server-side-model-master-detail/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

Normally parent rows (groups) can be expanded and child rows cannot be expanded (unless they are themselves parents). This means that normally only parent rows have expand / collapse icons.

For Master / Detail, expand and collapse icons are also needed at the master level. When doing Master / Detail, expand and collapse icons are also needed to expand the child rows where those rows are also master rows.

Rather than use the `autoGroupColumnDef` for the master rows as shown in the example above, simply specify a group cell renderer on the column that should show the expand / collapse icons.

This is shown in the code snippet below:

``` code
columnDefs: [
    { field: 'country', rowGroup: true },
    { field: 'accountId', maxWidth: 200, cellRenderer: 'agGroupCellRenderer' },

    // more column definitions
],
```

## Combining Tree Data with Master Detail <a href="https://www.ag-grid.com/javascript-data-grid/server-side-model-master-detail/#combining-tree-data-with-master-detail" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/server-side-model-master-detail/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

It is possible to combine [Tree Data](https://www.ag-grid.com/javascript-data-grid/server-side-model-tree-data/) with Master Detail in the Server-Side Row Model. This allows to display hierarchical (tree-structured) data, where each master row can expand to show a detail grid.

To enable this, set both `treeData: true` and `masterDetail: true` in your grid options. You will also need to provide the required callbacks for tree data:

- **isServerSideGroup(dataItem):** Returns `true` if the row is a group (has children).
- **getServerSideGroupKey(dataItem):** Returns the key for the group node (e.g., an ID).

The detail grid is configured using `detailCellRendererParams`, just as in other Master Detail scenarios.

Example configuration:

``` code
const gridOptions = {
    columnDefs: [
        { field: 'employeeName', cellRenderer: 'agGroupCellRenderer' },
        { field: 'jobTitle' },
        { field: 'employmentType' }
    ],
    rowModelType: 'serverSide',
    treeData: true,
    masterDetail: true,
    isServerSideGroup: dataItem => !!dataItem.children,
    getServerSideGroupKey: dataItem => dataItem.employeeId,
    detailCellRendererParams: {
        detailGridOptions: {
            columnDefs: [
                { field: 'project' },
                { field: 'duration' }
            ]
        },
        getDetailRowData: params => {
            params.successCallback(params.data.projects || []);
        }
    }
};
```

## Detail Row Height <a href="https://www.ag-grid.com/javascript-data-grid/server-side-model-master-detail/#detail-row-height" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/server-side-model-master-detail/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

The height of detail rows can be configured in one of the following ways:

1.  Using the `detailRowHeight` grid option property to set a fixed height for each detail row.
2.  Using the `getRowHeight()` grid option callback to explicitly set height for each row individually. This callback will need to work out the pixel height of each detail row.
3.  Using the `detailRowAutoHeight=true` property to let the grid automatically size the detail rows / grids to fit their rows.

The following snippets compares these approaches:

Option 1 - fixed detail row height, sets height for all details rows

``` code
const gridOptions = {
    detailRowHeight: 500,

    // other grid options ...
}
```

Option 2 - dynamic detail row height, dynamically sets height for all rows

``` code
const gridOptions = {
    getRowHeight: params => {
        const isDetailRow = params.node.detail;

        // not that this callback gets called for all rows, not just the detail row
        if (isDetailRow) {
            // dynamically calculate detail row height
            return params.data.children.length * 50;
        }
        // for all non-detail rows, return 25, the default row height
        return 25;
    },

    // other grid options ...
}
```

Option 3 - use autoHeight

``` code
const gridOptions = {
    detailCellRendererParams: {
        autoHeight: true,
    },

    // other grid options ...
}
```

<img src="out_ag_grid/server-side-model-master-detail/index_media/ae2971b6b482d433157d4aac53e206fe28cf84e4.svg" class="_icon_1jqgc_1 icon" />

Purging the cache and dynamic row heights do not work together for the Server-Side Row Model. If you are using dynamic row height, ensure `maxBlocksInCache` is not set.

## Lazy Loading Detail Rows <a href="https://www.ag-grid.com/javascript-data-grid/server-side-model-master-detail/#lazy-loading-detail-rows" class="_linkIcon_152wk_1 _docsHeaderIcon_152wk_8 button-secondary" aria-label="Heading link">Copy Link<img src="out_ag_grid/server-side-model-master-detail/index_media/76673052bb2634e0af37ce510b283b99cec64e64.svg" class="_icon_1jqgc_1 icon" /></a>

In the examples above, the data for the detail grid was returned with the master row. However it is also possible to lazy-load data for the detail row, see: [Providing Rows](https://www.ag-grid.com/javascript-data-grid/master-detail-grids/#providing-rows).

However note that detail rows will be purged once the master row is closed, or if the detail row leaves the viewport through scrolling. In both cases data will need to be fetched again.
