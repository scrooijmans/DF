Allow getting id from an object.

## <a href="https://opendal.apache.org/docs/rust/opendal/docs/rfcs/rfc_0293_object_id/index.html#motivation" class="doc-anchor">Â§</a>Motivation

Allow get id from an object will make it possible to operate across different operators. Users can store objectsâ€™ IDs locally and refer to them with different settings. This proposal will make tasks like backup, restore, and migration possible.

## <a href="https://opendal.apache.org/docs/rust/opendal/docs/rfcs/rfc_0293_object_id/index.html#guide-level-explanation" class="doc-anchor">Â§</a>Guide-level explanation

Users can fetch an object id via:

``` rust
let o = op.object("test_object");
let id = o.id();
```

The id is unique and permanent inside the underlying storage.

For example, if we have an s3 bucket with the root `/workdir/`, the objectâ€™s id `test_object` will be `/workdir/test_object`.

## <a href="https://opendal.apache.org/docs/rust/opendal/docs/rfcs/rfc_0293_object_id/index.html#reference-level-explanation" class="doc-anchor">Â§</a>Reference-level explanation

`id()` and `path()` will be added as functions of `object`:

``` rust
impl Object {
    pub fn id(&self) -> String {}
    pub fn path(&self) -> String {}
}
```

- `path` is a re-export of call to `Metadata::path()`.
- `id` will be generated by Operatorâ€™s root and `Metadata::path()`.
